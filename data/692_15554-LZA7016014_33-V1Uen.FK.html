class MaintenanceUserSecurity

MaintenanceUserSecurity

This MO provides an action for shutting down all other sessions than maintenance user sessions.
This is necessary in order to configure attributes in this MO as well as adding or removing MaintenanceUser MOs.
This MO also provides attributes for configuring login parameters of maintenance users.
Note:If the user has no login failures in the last FailedLoginAttemptPeriod or the user is not locked then the changes to the attributes FailedLoginAttemptPeriod,loginDelay,noOfFailedLoginAttempts and userLockoutPeriod take effect immediately, otherwise old values are used.

This MO is created by the system.



Actions
void
closeSessions ( );
This action will close all sessions except for maintenance user sessions.
string
unlockUser ( string maintenanceUserName );
This action will unlock a locked maintenance user.

Parameters

Name: maintenanceUserName

Description: Maintenance user's username for SSH-based login.See MO MaintenanceUser.username.

The maximum number of failed login attempts allowed, within failedLoginAttemptPeriod.

boolean

Whether management of MaintenanceUser MO's is allowed by authorized LDAP users.

When set to false authorized LDAP users can read MaintenanceUser MOs. Maintenance users can create, delete and modify MaintenanceUser MOs.

When set to true authorized LDAP users can create, delete and modify MaintenanceUser MOs. Maintenance Users can read MaintenanceUser MOs.
boolean

Whether logging in as a maintenance user is only possible when connection with LDAP fails.

When set to false user authentication is first performed towards maintenance user accounts and then with LDAP accounts.

When set to true user authentication is performed towards accounts defined in the LDAP server. If connection with the LDAP server fails user authentication is performed towards the maintenance user accounts.
