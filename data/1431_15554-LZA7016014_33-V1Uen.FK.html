IpsecProposalProfile

Common configuration parameters used by Child SAs.

Maximum number of IpsecProposalProfile MOs in node is limited to:
- For Baseband in RAN Compute Group 1, 2, or 3: 8,
- For Baseband in RAN Compute Group 4 or higher: 60.

ChildSaLifetime
childSaLifetime
Lifetime for negotiated Child SAs.
In order for node to complete the rekeying process before lifetime expires, node starts new rekeying according to soft lifetime. Soft lifetime is calculated as (0.8+0.1*R)*childSaLifetime, where R is random factor between 0 and 1.

Disturbances: Changing this attribute causes traffic disturbances.

ipsecProposal
Ordered list of Child SA proposals.

Each proposal contains set of cryptographic algorithms applicable for the Child SA.

If attribute is not set, system generates proposals based on node-supported algorithms.
Neither NULL encryption algorithm nor AES-GMAC integrity algorithms are included in this case.

First proposal in list has highest priority.

Disturbances: Changing this attribute causes traffic disturbances.
